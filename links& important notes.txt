Sir/Ma’am, I’m currently working on reproducing a research paper from AAAI 2025 conference. The paper is titled SRDC: Semantics-based Ransomware Detection and Classification with LLM-assisted Pre-training.
The main idea is: instead of just pattern-matching to detect ransomware, they teach a small language model (like a mini-GPT) to understand the meaning behind what the malware is doing — like a security expert would. They use both the meanings inside the program’s actions (API calls, file operations) and external knowledge (what experts say about suspicious behaviors).
I’m trying to run their open-source code on my laptop to see how it works in practice.
Right now on my laptop I have these main things:

A virtual environment folder called srdc
→ This is like a clean Python room where I installed only the packages needed for this project (PyTorch, Transformers, Pandas, etc.). So my main Python doesn’t get messed up.
The project folder: C:\DOMAIN-PRO-SRDC\project\srdc (cloned from GitHub)
Inside it there are these folders:
Feature_Internal_Semantic_Processing → contains the script Internal_Semantic_Processing.py
This is the most important first step. It takes raw ransomware behavior logs (API calls, registry changes, file opens) and rewrites them into natural English sentences.
Example: “GetFileSize” becomes “get file size”, and “FILES:OPENED:C:\file.doc” becomes “open the file C:\file.doc”.
This makes it easier for the language model to understand the intent.
Pretraining_Corpus → this has text files with explanations collected from Microsoft docs + generated by GPT-3.5 + checked by experts.
This is the “external knowledge” the model learns first (what APIs do, why some registry changes are dangerous, etc.). They used this to pre-train the model before fine-tuning on real data.
ZeroDay_Ransomware_Detection → script ransomware_0_day_detection.py
This is for detecting completely new ransomware families (zero-day) that the model never saw during training.
Ransomware_Family_Classification → script ransomware_family_classifier.py
This classifies which family the ransomware belongs to (like WannaCry, Locky, etc.).

data folder (I created it)
RansomwareData.csv → this is the main dataset from a 2016 research group (Sgandurra et al.). It has dynamic behavior logs of 582 ransomware samples (11 families) + 942 normal programs (goodware).
VariableNames.txt → list of all possible feature names (API names, registry keys, etc.) used in the CSV.


Right now I’m at the stage where I’ve downloaded the dataset, put it in the data folder, and I’m about to run the semantic processing script to convert the raw CSV into meaningful text that the model can understand better.
After that, I plan to run the detection and classification scripts using their pre-trained model from Hugging Face to see the accuracy numbers.
I’m doing this to learn:

How language models can be used in cybersecurity
How pre-training with domain knowledge helps in few-shot / zero-shot detection
Practical ML pipeline: data → preprocessing → model → evaluation

If it works well, I might try to test it on some dummy/benign programs or see if I can extend it a bit.
